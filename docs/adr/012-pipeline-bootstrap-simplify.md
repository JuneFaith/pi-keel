# ADR-012: Pipeline 拆解与 Bootstrap 简化（历史）

> 当前 Access Gate 结构以 ADR-017 和 ADR-018 为准。本记录保留旧 pipeline 与 bootstrap 简化的背景。

**分类：** 代码架构

## 问题

旧入口混合配置加载、命令注册、管道编排和 Shell 评估；bootstrap 注入状态也使用了超过实际需要的状态机。

## 决策

事件入口只负责注册和组装，策略逻辑按职责分层；原则内容移到独立的 `principles.md`；注入状态收敛为能表达实际语义的最小状态。

## 理由

职责分层便于独立测试和维护；原则内容与程序逻辑分离后无需修改 TypeScript 即可调整；最小状态减少重复注入和无效 handler。

## 不采用的方案

不继续在入口堆叠策略和注入内容，也不保留无法表达额外行为的状态机，因为两者都会扩大维护面而不增加能力。

## 当前状态

旧 pipeline 已由统一 Access Gate 取代；原则外置和入口负责组装的方向由 ADR-013 及当前源码继续采用。
