# ADR-016: 安全门控模块边界与统一命名（历史）

> 当前模块边界以 ADR-017 和 ADR-018 为准。本记录保留旧结构调整所确立的命名与职责原则。

**分类：** 代码架构

## 问题

旧安全门控把路径、权限、威胁检测、命令分类和生命周期状态混在一起，文件名也不能稳定表达职责。

## 决策

按领域职责拆分模块，使用表达领域概念的文件名；测试放在源码外并按被测职责命名；入口负责生命周期和组装，策略模块负责具体决策。

## 理由

清晰的职责和命名降低模块间耦合，使依赖方向、公共入口和测试边界可直接理解，同时避免为满足形式上的文件大小而制造低价值拆分。

## 不采用的方案

不继续使用泛化的 `utils`、`rules`、`detection` 等名称，也不按源码目录机械复制测试结构，因为这些做法无法表达真实职责。

## 当前状态

旧 phase/pipeline 结构已由 Profile access-gate 取代；职责边界、领域命名和源码外测试的原则仍适用。
