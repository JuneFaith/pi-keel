# ADR-011: 统一命令分类体系（历史）

> 当前命令分类和 Shell IR 以 ADR-017、ADR-018 为准。本记录保留“命令语义必须集中”的决策。

**分类：** 代码架构

## 问题

命令行为分散在多个模式列表中，新增或调整命令容易造成分类和安全策略漂移。

## 决策

以统一命令语义模块作为规则和查询的单一真相源，由受限 Shell 分析提取命令类别及文件操作意图，再交给路径策略和权限决策。

## 理由

命令规则、Shell 解析和路径意图集中管理，才能保证 `sudo`、重定向、wrapper 等不同表达方式进入同一决策链，并保持规则可测试。

## 不采用的方案

不在各个 gate、pipeline 或工具处理器中分别维护命令模式，因为重复列表会随修改产生语义漂移。

## 当前状态

旧 taxonomy 的 PLAN/BUILD 字段和目录已由 Profile shellPolicy、command-semantics 和 Shell IR 取代；单一真相源原则继续有效。
