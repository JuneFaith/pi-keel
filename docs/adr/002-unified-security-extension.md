# ADR-002: 统一安全扩展 vs 社区扩展并存

**分类：** 安全架构

## 问题

安全能力应由多个社区扩展分别提供，还是由一个扩展统一协调？

## 决策

创建统一的 `src/access-gate/` 扩展，集中处理 Profile、命令分类、路径策略、hard boundary 和审批。

## 理由

多个扩展会引入拦截顺序竞争、重复审批、分散配置和难以关联的审计信息。统一入口可以建立单一决策链，同时保留内部模块的独立测试边界。

## 不采用的方案

不并列安装多个社区安全扩展，因为它们的组合顺序和职责交集不可由 pi-keel 稳定控制。

## 影响

pi-keel 需要自行维护统一扩展与相关策略；社区扩展的独立更新不会自动纳入。
