# ADR-009: 移除 npm 发布、docs 目录和 AGENTS.md 模板

**分类：** 项目组织

## 问题

项目通过 Git 分发，不需要 npm 发布元数据；用户项目的 `AGENTS.md` 不应由 pi-keel 提供模板；使用文档也不需要额外嵌套入口。

## 决策

移除不必要的 npm 元数据和 `AGENTS.md` 模板，将使用文档放在根目录，并保留确有维护价值的架构、安全和待核查文档。

## 理由

每个文件都应有明确的维护对象和用户价值。减少发布配置与空目录可以降低分发和文档维护成本，同时不牺牲架构决策和安全边界记录。

## 不采用的方案

不保留未落地的规划文件或用户工程约定模板，因为它们要么重复过程信息，要么会越过用户项目的所有权边界。

## 影响

用户入口文档位于根目录；架构决策、安全边界和待核查事项分别位于 `docs/` 下的对应文件。
