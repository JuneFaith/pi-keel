# ADR-006: 统一命名规范

**分类：** 项目组织

## 问题

技能、目录和扩展名称的风格不一致，名称无法稳定表达用途和加载方式。

## 决策

采用分层命名规则：Foundations 使用简短描述名，Disciplines 使用名词短语，Workflows 使用动词-名词；复合名称使用 kebab-case，避免非必要缩写和人物名。

## 理由

名称应帮助模型和维护者推断技能用途；对自动匹配的 disciplines，名词短语比操作命令更适合描述方法论；统一格式也能减少配置和引用中的命名漂移。

## 不采用的方案

不继续混用动名词、生命周期命名或项目人物命名，因为这些风格不能稳定表达加载机制和技能职责。

## 影响

新增技能必须遵循所属层级的命名规则；改名时需同步目录、配置和引用。
