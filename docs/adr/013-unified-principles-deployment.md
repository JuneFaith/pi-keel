# ADR-013: 原则向用户项目的统一部署

**分类：** 文档架构

## 问题

通用工程约束、产物路径和状态转换分散在多个技能或 pi-keel 专属文件中，模型在用户项目中无法稳定取得同一份参考数据。

## 决策

采用“原则注入 + Quick Reference”双层部署：

- `principles.md` 注入全会话行为原则、Out of Scope 规则、产物路径、状态转换和用户项目 `CONTEXT.md` 结构。
- 技能文件只引用这些权威内容，不重复定义格式和规则。
- `survey-context`、技能文件和 ADR/溯源文档分别承担启动调查、按需流程和 pi-keel 自身决策记录。

## 理由

用户项目中可稳定获得的渠道只有会话注入内容和按需加载的技能。集中定义可以避免规则分叉、死链和重复维护。

## 不采用的方案

不把 pi-keel 的 CONTEXT、handoff、leading-words 或重复的流程细节作为用户项目的前置依赖；这些内容要么无法随安装部署，要么已有更合适的权威来源。

## 影响

`principles.md` 是通用参考数据的唯一注入来源；技能负责执行流程，ADR 和 `traceability.md` 负责记录 pi-keel 自身的决策与来源。模型路由、子 agent 协作和自动技能编排不属于本机制。
