# ADR-005: 技能三目录结构

**分类：** 项目组织

## 问题

技能目录应按领域、生命周期阶段还是加载方式组织？

## 决策

按加载方式分为三个目录：

- `foundations/`：始终激活的基础原则
- `disciplines/`：模型按上下文自动匹配
- `workflows/`：用户明确触发的工作流

## 理由

目录名直接表达技能何时生效，便于 pi 配置按目录启用或禁用，也比按领域或阶段组织更符合技能的实际加载机制。

## 不采用的方案

不采用按领域或生命周期阶段的目录，因为同一领域的技能可能有不同加载方式，目录无法表达这一关键差异。

## 影响

跨领域技能的归属可能不完全精确；具体触发条件由每个技能的 description 说明。
